# Огляд власних розробок (модулі Odoo)

Нижче наведено перелік розроблених мною кастомних модулів для Odoo, які розширюють базовий функціонал системи під конкретні бізнес-задачі.

### 1. Птахівництво (odoo_poultry)
Галузеве рішення для управління птахофермою. 
*   Охоплює весь життєвий цикл птиці: від посадки добового курчати до забою.
*   Дозволяє вести щоденний облік (Daily Logs): фіксація падежу, споживання кормів, вакцинації.
*   Автоматично розподіляє витрати та рахує точну собівартість кожної партії.
*   Містить кастомні дашборди (на базі OWL) для зручного введення даних персоналом на виробництві.

### 2. Планування оплат (payment_planning)
Модуль для управління фінансовими потоками та запобігання касовим розривам.
*   Створює візуальну чергу неоплачених рахунків із зазначенням пріоритетності.
*   Автоматично розраховує поточний денний баланс на основі залишків на банківських рахунках.
*   Контролює, щоб сума запланованих витрат не перевищувала доступний ліміт коштів.

### 3. Telegram AI Bot (telegram_ai_bot)
Інтеграційний модуль, який поєднує Odoo з месенджером Telegram.
*   Слугує шлюзом для виконання дій в ERP-системі через чат-бот.
*   Дозволяє керівництву дистанційно погоджувати фінансові операції (наприклад, затверджувати чергу платежів з модуля `payment_planning`) без необхідності входу в систему з комп'ютера.

### 4. Облік за договорами (ua_contract_simple)
Модуль для адаптації взаєморозрахунків під стандарти вітчизняної бухгалтерії (аналог логіки 1С).
*   Додає сутність "Договір" до документів продажу, закупівлі та платежів.
*   Дозволяє вести фінансовий облік та формувати акти звірки чітко в розрізі договорів, а не лише загалом по контрагенту.

### 5. Інструменти адміністратора обліку (accounting_admin_tools)
Спеціалізований набір функцій для головного бухгалтера та фінансового контролера.
*   Додає можливість примусової зміни дат у складських документах ("робота заднім числом").
*   Забезпечує механізм "Відновлення послідовності" — автоматичний перерахунок собівартості товарів (AVCO/FIFO) після внесення змін у минулих періодах.

### 6. Технічні переклади (translations_technical)
Допоміжний модуль для локалізації інтерфейсу.
*   Дозволяє підміняти стандартні системні терміни та підказки Odoo на звичний корпоративний словник клієнта без втручання в базовий код системи.
